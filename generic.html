Sikker Software

Dette er min sektion for Hvad min plan er med Sikker Software

Uge 35-36

Delmål Hvordan Hvornår er det opfyldt Evaluering
Lære noget basic viden omkring sikker software
_____________
 Følg et kursus i software sikkerhed(youtube eller andre medie resurcer fra en pålidlig kilde) Når jeg kan forklare noget omkring hvordan man laver en sikker software både kode mæssig og forbyggende(systemudvikling) Vi har fundet frem til vi gerne vil kigge ind i Risiko modeller og CIA modellen for at få noget sikker software ind i vores system.

Uge 37-38

Delmål Hvordan Hvornår er det opfyldt Evaluering
Finde ud af hvordan vi kan bruge sikker software i vores produkt
_____________
 Snakke med vejleder omkring hvordan vi kan bruge sikker software i vores system Når vi har noget vi kan kigge dybere ind i med sikker software Vi har fundet ud af CIA ikke er den bedste model for os da vi ikke har nogen "Users" men noget mere hacker beskyttende.

Uge 39-40

Delmål Hvordan Hvornår er det opfyldt Evaluering
Finde ud af hvordan man egentlig kan blive hacket
_____________
 Se video om hvordan folk er blevet hacket Når jeg har en forståelse for, hvordan kan man blive hacket og hvad er egentlig, sensetivt data man kan finde på personer som man kan bruget til at hacke dem. Jeg har fundet ud af at hacke ikke kun har med computere at gøre men også er meget, om man er blevet targeted, eller givet sensetivt data ud til nogen om sig selv, som man kan bruge til at blive hacket.

Uge 41-42

Delmål Hvordan Hvornår er det opfyldt Evaluering
Finde ud af hvad har vores projekt helt præcis brug for, for at være sikker software?
_____________
 Analysere vores system og finde ud af hvilke weakpoints vi har. Når vi ved vores system er sikkers Vi har fundet ud af at vores produkt er super godt i form af vi ikke har noget sensetivt data som kan gøre Vores kunde vil komme i problemer

Uge 43-45

Delmål Hvordan Hvornår er det opfyldt Evaluering
Vi skal gøre vores system så sikkert som muligt, ikke kune fra et hacker point men fra et punkt af hvis det sker hvordan fikser vi det. Vi skal analysere hvordan vores system vil håndtere forskellige situationer og sørge for at hvis noget går galt at det så kan fikses hurtig. Det er oplydt når vi har en Risiko analyse af vores produkt så vi kan sige at vores software er sikker.
